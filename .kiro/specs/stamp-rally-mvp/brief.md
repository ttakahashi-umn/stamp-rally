# Brief: stamp-rally-mvp

## Problem
IMSグループ約25,000人の職員向け70周年記念イベントにおいて、催し物の参加体験が弱く、会場回遊とイベント満足度の向上につながる導線が不足している。

## Current State
現状はスタンプラリーのデジタル実装がなく、参加導線・認証・押印・進捗可視化・不正抑止の仕組みが未整備。

## Desired Outcome
職員が配布メールのリンク/QRからすぐに参加開始でき、会場QRを読み取るだけで簡単にスタンプを獲得できる。結果として回遊行動を促進し、イベント満足度向上の土台を作る。

## Approach
メール配布QR/リンクによる初回認証と端末記憶を採用し、参加者はモバイルWebで会場QRを読み取って即時押印できる方式を採用する。  
不正抑止として、会場緯度経度に対するジオフェンス判定に加え、QRトークンの短TTL署名検証（リプレイ対策）をMVPに含める。  
実装は既存前提の React + FastAPI + SQLite を基本とし、ピークアクセスの同時書き込みは負荷試験で確認する。

## Scope
- **In**:
  - メール配布リンク/QRからの初回認証
  - 端末記憶による再訪時の簡易アクセス
  - 会場QR読み取りによるスタンプ付与
  - 会場マスタ（緯度経度含む）に基づくジオフェンス判定
  - QR短TTLトークン検証による基本的な不正抑止
  - 参加者向けスタンプ一覧・達成状況表示
- **Out**:
  - 企業SSO/ID基盤との本格連携
  - ネイティブアプリ（iOS/Android）
  - 高度な不正検知（端末指紋の厳密照合、機械学習検知）
  - 景品交換/抽選/ポイント外部連携
  - 詳細な満足度改善施策の自動提案機能

## Boundary Candidates
- 認証・セッション境界（メール配布情報、初回認証、端末記憶）
- 押印境界（QR読み取り、署名検証、スタンプ付与処理）
- 不正抑止境界（ジオフェンス判定、時刻検証、イベント期間制御）
- 参加者体験境界（起動導線、一覧表示、進捗可視化）

## Out of Boundary
- 70周年イベント全体企画の改善コンサル
- 参加者満足度施策の運用設計そのもの
- 全社向け長期ポイントプラットフォーム化

## Upstream / Downstream
- **Upstream**:
  - 参加者への事前メール配布オペレーション
  - 会場ごとのQR設置場所および緯度経度データ整備
  - イベント運営側の会場・期間定義
- **Downstream**:
  - 回遊率/参加率の分析ダッシュボード拡張
  - 追加イベントへの再利用（周年以外の社内施策）
  - 景品・特典連動機能の将来追加

## Existing Spec Touchpoints
- **Extends**: なし（新規spec）
- **Adjacent**: `frontend`（React UI）、`backend`（FastAPI API/SQLite）

## Constraints
- 対象ユーザーは約25,000人規模
- モバイルWeb中心で、ブックマークから素早く起動できること
- 個人情報・位置情報の取り扱いに関する社内ルール準拠
- イベント期間内にMVPを実運用可能なスコープに限定すること
